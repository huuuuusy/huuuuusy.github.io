1 Introduction

As a very important research method in machine learning and well developed deep neural network, convolutional neural network has several important applications in practical problem solving. With different application contexts, CNN has derived a number of variations and achieved good results in recent years.

2 Original CNN Model

The earliest CNN can be regarded as a two-dimensional neural network without hidden layers. The input layer and the convolution kernel are simply convolved, and then the feature extraction is completed to obtain the network output layer. For example, LeNet uses the single convolution + single pooling structure, in which convolution layer performs feature extraction and the pooling layer completes down sampling. Subsequently, AlexNet uses a multi-convolution + single-pooling structure to extract richer features. VGG , which based on the same core structure, is applied to large-scale image recognition as well. However, the CNNs mentioned above use only one fixed-size convolution kernel per layer and therefore belong to overlay network. For instance, VGG uses a large amount of convolution layers in its model.

3 Multi-dimension Convolution Kernel – Inception Structure and Variants

3.1 Multi-dimension Convolution Kernel – Inception Structure

The multi-hidden-layer with non-linear structure is an improvement over the traditional multi-layer network, which aims to enhance non-linear expression by adding hidden layer. It adopts Inception structure, which uses multiple convolution kernels of different sizes to obtain different scales of features in the same convolution layer, then combines the features to output better training effects. Figure 1 shows the original Inception structure. The input features are processed simultaneously with the 1X1, 3X3 and 5X5 convolution kernels to obtain the calculated results respectively, then these calculated results are combined to obtain better features.

In 2014, Christian Szegedy proposed GoogLeNet based on Inception and achieved good results in large-scale visual recognition. In 2015, he added Batch Normalization to Inception and replaced the 5X5 convolution kernel with two 3X3 convolution kernels to obtain InceptionV2. By splitting the previously used 7X7 convolution kernel into a 7X1+1X7 structure, he improved the accuracy and obtained InceptionV3 in the same year. In 2016, Christian Szegedy proposed a better model–InceptionV4 by adding a residual structure.

3.2 Reduce the Number of Convolution Parameters–Add 1X1 Convolution Kernel

Christian Szegedy found that simply introducing multi-size convolution kernels may add numerous extra parameters, while adding several 1X1 convolution kernels can reduce the amount of parameter. The revised Inception structure is shown in Figure 2.

If the input and output dimensions are all 256 dimensions, the original situation shown in the left of Figure 3 is calculated by a 3X3X256 convolutional layer and the number of parameters involved in the calculation is: 256X3X3X256=589824 . In the structure shown on the right, the input 256-dimensional matrix passes through 1X1X64, 3X3X64, 1X1X256 convolutional layers and outputs 256 dimensions. The number of parameters involved in the calculation is: 256X1X1X64+64X3X3X64+64X1X1X256=69632, which reduces an order of magnitude.

3.3 (Extremity) Multi-dimension Convolution Kernel – DepthWise Separable Convolution

DepthWise Separable Convolution is a continuation of Inception structure. Figure 4 shows two equivalent expressions of primary Inception structure, the input is convolved with 1X1 convolution kernel and divided into three parts, then convolved with 3X3 convolution kernel respectively and combined together to get the output. To further increase the number of cuts, it is possible to cut the input to the same number as the output channels. Inception structure for 7 output channels is shown in Figure 5, this extremity version of Inception structure is called DepthWise Separable Convolution.

Xception model mentioned in CVPR2017 used a similar structure. At first, each channel performs its own convolution operation and get feature maps, then accomplished 1X1 pointwise convolution of feature maps to complete the fusion of separation channels as shown in Figure 6. This operation exceeded the performance of InceptionV3 in the ImageNet 1000 category task and reduced the number of parameters at same time.

Xception outputs one feature map (basement layer) per channel, while Min Wang proposed a multi-basement-layer optimization approach in 2016. He designed three optimization methods including topology to improve the accuracy and reduce the number of operations.

4 Group convolution and Variants

From a practical point of view, deep neural networks often require massive storage and computational resources due to the limitations of huge matrix operations. Reducing the redundancy of convolutional units in neural networks is one of the important studies to solve this problem, and is divided into spatial and channel directions.

4.1 Original Group Convolution——AlexNet

Group convolution first appeared in AlexNet and the structure is shown in Figure 7. At that time, convolution cannot be handled on the same GPU in training process due to the limited hardware resources. Therefore, the author divides the feature maps into multiple GPUs and processes them separately then merged the results.

4.2 Random Grouping Scheme 1 – ShuffleNet

Group convolution of AlexNet divides the characteristic channels into different groups evenly and finally fused features by two fully connected layers. Therefore, the features between different groups can only be merged at the last moment, which is not conducive to model generalization. In order to solve this problem, ShuffleNet performs channel shuffle before each group convolution, and these shuffled channels are allocated to different groups. After a group convolution, a channel shuffle is performed again, and these channels are assigned to the next layer, then continue to cycle. The principle of channel shuffle is shown in Figure 8. After channel shuffle, the output characteristics of group convolution can contain more channels and thus have a better representation.

In addition, the group convolution of AlexNet is actually a standard convolution operation, while ShuffleNet’s group convolution operation is DepthWise convolution. Thus ShuffleNet, which incorporates channel shuffle and DepthWise convolution, obtains small parameters with high accuracy.

4.3 Random Grouping Scheme 2 – Interleaved Group Convolution

The idea of Interleaved Group Convolution (IGC) proposed by Microsoft Asia Research Institute is derived from the concept of Deep Fusion. The essence of Interleaved Group Convolution is to merge the networks of different branches in middle layer. Each IGC module includes two group convolution processes – the primary group convolution and the secondary group convolution, as shown in Figure 9.

Group convolution is used in AlexNet to distribute the model across two GPUs to address memory handling issues. Taking a 32-channel model as an example: the 32 input channels are divided into four groups and each group has eight channels, then the four groups are separately convolved and this process is the group convolution. The advantage is that fewer parameters can increase computational speed, but different sets of output channels are not relevant to the input channel since there is no interaction between each set of convolutions. In order to correlate the input channels, IGC introduces the second group convolution cleverly. During the second group convolution, the input channels of each group come from different groups in the first group convolution process to achieve interleaved complementarity.

From the viewpoint of eliminating convolution kernel redundancy, one group convolution is equivalent to an ordinary convolution with sparse kernels, while the interleaved group convolution is equivalent to the ordinary multiplication of two sparse kernels convolution. The multiplication of these two sparse matrices leads to a two-by-two dense matrix, which is a linear process. Under the condition of the same network model and the same computational complexity, the network using the IGC module has a wider network and better performance than the ordinary convolution.

5 Solve Gradient Vanishing – Skip Connection

The original CNN increases the network depth by overlaying convolution layers to improve the recognition accuracy. However, too many layers of convolution will appear gradient vanishing, which means that the gradient cannot be transferred to the previous network layer effectively and the previous layer parameters cannot be updated. In 2015, Kaiming He proposed Deep Residual Learning, which solved the gradient vanishing problem through skip connection. As shown by the orange dotted line in Figure 10, the gradient in back propagating is 1. It passes through an addition gate and back to the previous layer through the skip connection shown by the dashed orange line. After that, the gradient changes to [1, 0.0001, 0.01], which has one more “1” than the previous gradient. This method solves the order of magnitude gradient declines when back to previous levels. It is because of the skip connection that the gradient from the deep layer passes directly and back to the previous level, allowing shallow network layer parameters to be effectively trained.

ResNet proposed by Kaiming He uses skip connection to achieve a smooth, unimpeded passage of each Res Block, which enables the construction of a thousand-layer residual network. The ResNet structure is shown in Figure 11.

The skip connection of DenseNet proposed by CVPR2017 Best Paper Award not only connects the upper and lower layers, but also achieves cross-layer connection, which makes the gradient of each layer is derived from several previous layers. The structure of DenseNet is shown in Figure 12. When increase the depth of the network, DenseNet increases the width of each dense block as well, which can improve the ability of feature recognition. What’s more, the amount of parameter that need to be calculated is greatly reduced since the horizontal structure of the dense block is similar to that of the Inception block.

6 Increase Receptive Field of Convolution Kernel – Dilated Convolution

Dilated Convolution is another variant of CNN, which aims to allow a fixed-size convolution kernel to obtain a wider range of data during convolution, then increases the amount of information contained in the convolution output. Figure 13 shows a Dilated Convolution structure proposed in 2015. Taking the 3X3 convolution kernel as an example, (a) is1-dilated convolution and consistent with the general convolution calculation. (b) is the 2-dilated convolution，for the 7X7 region in (b), only 9 red points are convolved with the 3X3 convolution kernel and the rest points are ignored. (c) is the 4-dilated convolution，for the 15X15 region in (c), only 9 red points are convolved with the 3X3 convolution kernel and the rest points are ignored.
Traditional CNN in image processing always convolves the input information then pooling to reduce the image size as well as increasing receptive field. However, since the image size is reduced during pooling, the image needs to be recovered by down sampling before prediction, which may result the loss of image information. While according to the principle of Dilated Convolution, it can increase the receptive field without pooling, which avoids the information loss in the pooling process. Therefore, Dilated Convolution performs well in image processing problems which require global information (such as image segmentation), or in natural language processing issues that depend on longer sequence information (such as WaveNet for speech synthesis and ByteNet for machine translation).

7 Change Shape of Convolution Kernel – Deformable Convolutional Networks

Deformable Convolutional Networks believes that the regular shape of the convolution kernel (such as the general use of the square 3X3 convolution) may limit the feature extraction. If the convolution kernel can be transformed to suit the error during the back propagation, the network can extract features better. Figure 14 (a) is the original convolution kernel; (b) indicates that a convolution kernel learns an offset during training process, then obtains a new convolution kernel; convolution kernels may expand as shown in (c) for identifying objects of different scales; if rotating object recognition is performed, the convolution kernel may be rotated as shown in (d).

The principle of Deformable Convolutional Network proposed in 2017 is shown in Figure 15. First, the input feature map is convoluted to obtain the output (offset field), then calculate to get the new offsets. After that, it uses the new offsets to complete the convolution and obtain the output with better feature recognition.

8 Feature Recalibration – Squeeze-and-Excitation

SENet, Image Classification Champion Model in ImageNet 2017 Contest, proposes an important method to distinguish features by making improvements in feature dimensions. SENet automatically learns how important each feature channel is, then increases the useful features in accordance with the calculated importance and suppresses useless features. Figure 16 is the schematic of the A Squeeze-and-Excitation Block. A set of features is output on the previous level, then divided into two routes. The first one passes directly, and the second one first performs a Squeeze operation (Global Average Pooling), which compresses the two-dimensional features of each channel into a one-dimensional to get a feature channel vector (each number represents the characteristics of the corresponding channel). After that, do Excitation operation to input this series of eigenvector in two full connection layers and sigmoid, then model the correlation between characteristic channels to get the weight of each channel. Finally, add these weights to original features by using the Scale multiply channel (the first path) to complete the weight assignment of feature channel. In addition, the article mentions that the SENet can be combined with other mainstream network structure. The combination of Inception and SE-Inception Module is shown in Figure 17.

9 Summary

CNN evolves from the mega network to the lightweight network step by step, and the accuracy of the model is higher and higher. The current CNN models are mainly focused on trading off between speed and accuracy, hoping the model can be fast and accurate. Therefore, from the original AlexNet and VGG model to a small volume of Inception and Resnet series, then to the current MobileNet and ShuffleNet model, the tendency of CNN can be summarized as follows:

9.1 Convolution Kernel

As an earlier CNN, AlexNet uses large-size convolution kernels to expand the receptive field. Large-size convolution kernel will increase calculation, which is not conducive to increasing the depth of model and reduces the computational performance. Inception verifies that the training effect of two 3X3 convolution kernels is better than one 5X5 convolution kernel. What’s more, the parameters of two convolution kernels are 19 (3X3X3+1), less than one 5X5 convolution kernel (5X5X1+1=26).

Therefore, in convolutional network design, large-size convolution kernel should be avoided as much as possible, and single-size convolution kernel can be replaced by multi-size convolution kernel.

In practical problems, the fixed-shape convolution kernels can be replaced by deformable convolution kernels according to the characteristics of the problem to be solved.

In large-scale network design, 1X1 convolution kernel can be used properly to reduce network computation.

9.2 Convolutional Channel

Standard convolution can be replaced with DepthWise convolution, and grouping convolution can be used to improve computational efficiency during big data training.

Before group convolution, it’s better to use channel shuffle to improve the representativeness of the output features.

9.3 Convolutional Connection

For designing deep neural networks with more layers, using skip connection to solve gradient vanishing is a good choice.

In addition, densely connection can be used to achieve feature fusion according to DenseNet, which is very useful to enhance the training effect of network.
