By the way, I hope you can forgive my poor English skill:)

Preface

There are two important steps for building a personal website.

The first is buying a special domain name such as yourname.com. You can get your domain in different Internet service providers, like GoDaddy, Alibaba Cloud and Tencent Cloud. I will use GoDaddy as an example to introduce detailed procedures since my domain is bought in this website. (Based on the cost consideration, choosing other providers may be a better choice and you can get detailed gudiance by some related blogs.)

For second step, you need to deploy your personal website in a space. Choosing a cloud host seems good, but it is difficult for novice to practice. In fact, Github provides a space(300M) for users to deploy their website, and the only thing that you need is just a Github account. Readers who do not need personal domain can start from step two, since Github will provide you a domain like https://github.com/your_name/your_project_name.

Let’s start now!

I. Buy a domain

It is very convenient to buy a domain in GoDaddy, and here are specific steps:

1. Search a domain that you want

P.S. You can choose Singapore as location in the upper left of this page and browse it in simplified Chinese version.

2. Add to shop cart

3. Pay for it

P.S. Dodaddy will show you a page to make sure if you need the extra service before you pay for the domain. It’s better to choose No.Thanks even if the service is not expensive since what we need is just a domain.

Now you can choose the term that you need(1/2/3/5/10 years) and pay for it. 3 years may be a good choice because you can renew this domain before the deadline if you want to keep using.

P.S. You can search GoDaddy promo code and get a discount by using the newest prommo code.

Before paying the bill, you need to login an account. Some necessary information is needed, and for successful payment, you’d better to fill it with the detailed personal information. For Chinese users, you can find solutions in Dute.me if you have any question for paying with Alipay.

4. Check your domain

After you paying for your domain, reLogin and check it in your account.

What’s more, you will receive an Email to verify your account, and remember to check it.

II Download some software

P.S. Images in this part are from beiyuu‘s blog.

1. Install Node.js and Git

Before deploying the website, we need to download two softwares.

Node.js

Git

I will skip the details since it is easy to install these softwares if you download the correct version.

2. Login Github

Access github and create your account.

P.S. Pay attention to your username and Email address, since Github always uses email to send important notification.

3. Configurate Github

Now we need some processes to connect the local project with remote Github.

Check SSH keys

For Windows, you can find Git Bash in the start menu and open it as a terminal, then type the instruction.

$ cd ~/.ssh

If you are new to git, it will shows No such file or directory, and you can skip next step.

Remove old SSH keys (Optional)

If there is an old key in PC, you need to back-up and remove it.

$ ls
$ mkdir key_backup
$ cp id_rsa* key_backup
$ rm id_rsa*

Generate the new key

$ ssh-keygen -t rsa -C "youremail@youremail.com" 
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<ENTER>

P.S. You need to type your email address in the first line, and just press ENTER to choose the default path.

Then you need to type a passphrase:

Enter passphrase (empty for no passphrase):<just enter a series of numbers that you like>
Enter same passphrase again:<enter again>

You will see an interface like this, which means you are successful to set the ssh key.

Add SSH key to Github

Open id_rsa.pub and copy the text correctly.

P.S. The reference path is C:\Users\usersname.ssh\id_rsa.pub. You need to show hidden files if you cannot see this file in the folder.

Login Github and click Account Settings–>SSH and GPG keys–>New SSH Key, then paste your SSH in the textbox and click Add key.

Test

Now we can type this instruction in Git to check weather it is success.

$ ssh -T git@GitHub.com

P.S. Do not change the email address in this instruction.

Then you can get a feedback like this:

The authenticity of host 'GitHub.com (207.97.227.239)' can't be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

Now type yes and you will see:

Hi huuuuusy! You've successfully authenticated, but GitHub does not provide shell access.

Set user’s information

Some personal information is needed to replenish in Git. Since user name and email will be used to record your submit and authorize management, it is better to type your true name in this instruction.

$ git config --global user.name "your_name"
$ git config --global user.email "your_email@youremail.com"

Now our PC have connected with Github successfully.

P.S. For any questions in setting SSH keys, you can consult these two pages:

GitHub Help - Generating SSH Keys

GitHub Help - Error Permission denied (publickey)

III Build a static blog frame

Other articles may have different order to operate, and I will choose a convenient one to introduce. What’s more, huuuuusy is my user name in Github, you should change it to your name in the following steps.

1. Introduction of Hexo

As the offical website advertises, Hexo is a fast, simple&powerful blog framework and you only need few commands to deploy your site to Github Pages. Plentiful themes are available in it, which are convenient to be changed or modified for personal demands.

2. Install Hexo

You can consult the guide in offical website, and install Hexo with npm in Git:

$ npm install -g hexo-cli

Now, consult guide and operate step by step to setup Hexo. First, create a folder in PC, and rename it as huuuuusy.github.io. Then, open Git Bash in this folder and tpye:

$ npm install

Once initialised, your folder will look like:

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

You can design personal interface by change parameters in these files, and I will introduce this part in next blog, now just ignore these details. If you want to get more information in this part, please browse the offical website.

3. Install a theme(optional)

Hexo provides plentiful themes for users, and you can choose what you like in Hexo theme. For novice, Next may be a good choice since it is the most popular theme in Hexo with numerous guides and you can configure your personal interface by consult these articles.

The defaul theme is landscape, you can choose changing it to other themes by Googling different guides now or changing it after you have finished all necessary procedures. In next blog, I will introduce the specific process about design personal theme, so just skip this step now:)

4. Deploy Hexo

We can test the original interface of Hexo by this commond:

$ hexo g
$ hexo s

Now open the Explorer and type http://localhost:4000 in the address bar, you will see an interface like the theme that you choose in step 3.

By the way, after you changing some parameters to optimize the interface, you can use above commond to preview the effect.

IV Deploy the blog frame to GitHub Pages

Now open GitHub and create a new repository which name is huuuuusy.github.io since I’ve build a folder with the same name in part III. (Of course, you should change it to the name that you’ve set in the local folder.)

Open Git Bash and tpye commands as follow:

$ git add . 
$ git commit -m "first post" 
$ git remote add origin https://github.com/username/projectName.git 
# We need this command to push the local change to GitHub Page. In fact, you should change the URL as yours, like https://github.com/huuuuusy/huuuuusy.github.io.git, and you may need to type user name and password after that step
$ git push origin gh-pages

After that, we need to add some necessary information in -config.yml to finish the deployment.

Find the keyword “deploy” in this file and change it as follow:

deploy:
  type: git
  repository: https://github.com/huuuuusy/huuuuusy.github.io.git
  branch: master

Then open Git and type the following commands to complete deployment:

$ hexo g
$ hexo d

Now type the huuuuusy.github.io in the browser(need change it to yours), you’ll see a webpage like you’ve seen in http://localhost:400.

2. Set GoDaddy

Now come back to GoDaddy and change the default name servers to DNSPod.

Click manage DNS and change the default value to f1g1ns1.dnspod.net and f1g1ns2.dnspod.net.

3. Set GitHub repository

Open GitHub and create a new file names CNAME in huuuuusy.github.io, then recodes your domain in it.

Ok, wait for a few minutes to resolve the domain, then you can access your website by huuuuusy.github.io or hushiyu1995.com.
