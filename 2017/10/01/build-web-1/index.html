这篇博客将介绍如何搭建一个拥有个性域名的博客网站。

对于搭建个人网站，有两个重要的步骤：

  1. 购买一个个性域名，如yourname.com. 可以选择不同的互联网域名提供商，如GoDaddy, Alibaba Cloud 和 Tencent Cloud.本文以GoDaddy作为例子来介绍我的个人网站http://hushiyu1995.com/的搭建流程。

  2. 此外，需要将个人网站部署到服务器上。云服务器是很不错的选择，但是对于新手来说难度比较大。本博客选择
    Github，其提供了300M的免费空间共用户部署个人博客网站，并且只需要Github账户就可以注册。不需要个性域名的读者可以跳过接下来的第一部分和第二部分，因为Github已经提供了形式如 https://github.com/your_name/your_project_name的域名供用户使用。

I. 购买域名

在GoDaddy上购买域名非常方便，具体步骤如下：

1. 选择喜欢的域名

P.S. 如果想使用简体中文版，请将地区选择为新加坡。

*2. 添加到购物车

3. 购买

P.S. Godaddy会询问是否需要附加服务，从节省费用的角度考虑，请选择No.Thanks，因为附加的其他域名对搭建博客并没有任何帮助。

然后可以选择购买域名的期限，我选择的是3年的方案，在到期前也可以手动续费来延长使用期限。

P.S. 为了省钱，也可以自行搜索 GoDaddy promo code 以获取最新的优惠码。

在结账前需要登陆自己的账户并完善个人信息。对于中国用户，可以参考Dute.me 来使用支付宝完成付款流程。

4. 检测域名

在结账后，重新登陆并检测自己的账户。

此外，Godaddy也会发送一份验证邮件，请注意验证自己的账户信息。

II 软件安装

P.S. 本部分的所有图片来自于beiyuu的博客。

1. 安装Node.js和Git

在部署之前，需要安装如下两个软件工具。

Node.js

Git

请参照官网教程安装，步骤很简单。

2.登陆Github

登陆github并创建自己的账户.

P.S. 注意用户名和Email信息，这些信息将对随后的验证和密码找回非常关键。

3.配置Github

本步骤需要将个人本地项目和Github代码仓进行远程连接。

检查SSH keys

对于Windows用户，可以使用Git Bash作为终端进行操作，输入如下指令：

$ cd ~/.ssh

如果是第一次使用git,终端将显示No such file or directory，可以直接跳过进入下一步。

移除旧的SSH Keys

如果电脑里有之前版本的SSH keys，需要手动使用如下指令进行移除。

$ ls
$ mkdir key_backup
$ cp id_rsa* key_backup
$ rm id_rsa*

生成新的key

$ ssh-keygen -t rsa -C "youremail@youremail.com" 
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<ENTER>

P.S. 需要输入个人的邮箱在第一行命令中，然后按Enter选择默认设置。

然后需要输入密码:

Enter passphrase (empty for no passphrase):<just enter a series of numbers that you like>
Enter same passphrase again:<enter again>

随后将获得如下图所示的SSH key.

将SSH key添加到Github

打开id_rsa.pub并且复制.

P.S.参考路径是C:\Users\usersname.ssh\id_rsa.pub. 需要显示隐藏文件才能看到该文件。

登陆Github然后选择Account Settings–>SSH and GPG keys–>New SSH Key, 粘贴key的信息并点击Add key.

测试

现在测试是否配置成功。

$ ssh -T git@GitHub.com

P.S. 在这一步中，不要修改邮箱地址。

然后会得到如下的反馈:

The authenticity of host 'GitHub.com (207.97.227.239)' can't be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

选择yes后:

Hi huuuuusy! You've successfully authenticated, but GitHub does not provide shell access.

设置用户信息

下面需要添加个人信息。

$ git config --global user.name "your_name"
$ git config --global user.email "your_email@youremail.com"

完成之后，电脑就和github完成绑定.

P.S. 如果配置SSH Key出问题，可以参考如下网页进行解决:

GitHub Help - Generating SSH Keys

GitHub Help - Error Permission denied (publickey)

III 搭建静态博客框架

请注意，huuuuusy是我的github用户名，以下的所有步骤涉及到这个名字的部分都要进行相应替换。

1. Hexo介绍

Hexo提供一套快速部署静态博客的方案，只需要几步操作就可以将自己的网站和github page进行绑定。此外，Hexo还提供大量的主题可以直接使用。

2. 安装Hexo

参考官方教程并在git中利用npm完成安装:

$ npm install -g hexo-cli

然后参考官方教程来初始化Hexo。

首先，在电脑上创建huuuuusy.github.io文件夹。在文件夹中打开Git:

$ npm install

一旦完成安装，文件夹下文件目录将如下图所示:

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

如果想设计个人的界面，可以通过修改文件参数完成。网上有很多Hexo主题修改的教程，可以根据教程修改默认主题，完成个性化配置。

3. 安装主题

Hexo提供大量精美主题，可以在Hexo theme进行挑选. 对于新手, Next是不错的选择，这是Hexo最热门的主题之一，大量有关主题配置和修改的教程都可以在作者的官网上找到。

Hexo的默认主题是landscape,也可以在完成所有安装步骤后再进行修改。

4. 部署Hexo

通过下面的指令测试默认的界面:

$ hexo g
$ hexo s

然后在浏览器输入http://localhost:4000查看，显示的界面是在上一步中挑选的主题界面。

在每次完成对主题的修改后都可以用上面的指令预览修改效果。

IV 将Hexo部署到Github Page上

在Github新建huuuuusy.github.io 的项目，此处项目应该和之前在本地建立的文件夹名字相同。

在git输入:

$ git add . 
$ git commit -m "first post" 
$ git remote add origin https://github.com/username/projectName.git 
# We need this command to push the local change to GitHub Page. In fact, you should change the URL as yours, like https://github.com/huuuuusy/huuuuusy.github.io.git, and you may need to type user name and password after that step
$ git push origin gh-pages

然后需要在-config.yml添加一些必要信息以完成部署。

找到文件中”deploy”的关键词，并修改为:

deploy:
  type: git
  repository: https://github.com/huuuuusy/huuuuusy.github.io.git
  branch: master

然后在git中输入:

$ hexo g
$ hexo d

在浏览器中输入huuuuusy.github.io，应该可以看到和http://localhost:4000相同的界面.
